# 微信小程序网络请求配置指南

## 问题描述
在微信小程序中请求 `http://shop.dayu.club:8199/frontend/user/login` 接口时出现 `net::ERR_EMPTY_RESPONSE` 错误。

## 解决方案

### 1. 开发环境配置（推荐）
在微信开发者工具中，确保已进行以下配置：

#### 修改 `project.config.json`
```json
{
  "setting": {
    "urlCheck": false,  // 关闭域名校验
    // 其他配置...
  }
}
```

#### 修改 `app.json`
```json
{
  "requestLegacyDomain": true,  // 允许使用非HTTPS域名
  // 其他配置...
}
```

### 2. 生产环境配置
对于生产环境，必须进行以下操作：

#### 在微信公众平台配置
1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入「开发」->「开发设置」
3. 在「服务器域名」中添加：
   - request合法域名：`https://shop.dayu.club`
   - uploadFile合法域名：`https://shop.dayu.club`
   - downloadFile合法域名：`https://shop.dayu.club`

#### 服务器要求
- 必须使用HTTPS协议
- 必须备案的域名
- 必须支持TLS 1.2及以上版本

### 3. 本地测试方法

#### 方法一：使用测试页面
访问 `/pages/test/test` 页面，点击"测试网络连通性"按钮，检查基础网络是否正常。

#### 方法二：检查开发者工具设置
1. 打开微信开发者工具
2. 点击右上角「详情」
3. 选择「本地设置」
4. 勾选「不校验合法域名、web-view域名、TLS版本以及HTTPS证书」

### 4. 常见问题排查

#### 问题1：net::ERR_EMPTY_RESPONSE
**原因**：服务器没有返回响应或连接被拒绝
**解决方案**：
- 检查服务器是否正常运行
- 检查防火墙设置
- 检查端口8199是否开放

#### 问题2：域名未配置
**原因**：域名未在微信公众平台配置
**解决方案**：
- 开发环境：关闭域名校验
- 生产环境：在公众平台配置合法域名

#### 问题3：HTTPS要求
**原因**：生产环境必须使用HTTPS
**解决方案**：
- 部署SSL证书
- 使用HTTPS协议

### 5. 开发建议

1. **开发阶段**：使用 `urlCheck: false` 关闭域名校验
2. **测试阶段**：使用测试域名进行功能测试
3. **上线前**：配置正式的HTTPS域名
4. **错误处理**：使用统一的请求封装，便于错误排查

### 6. 紧急解决方案

如果急需测试，可以使用以下临时方案：

```javascript
// 在app.js的onLaunch中添加
wx.setEnableDebug({
  enableDebug: true
})
```

然后在开发者工具中开启「不校验域名」选项。
