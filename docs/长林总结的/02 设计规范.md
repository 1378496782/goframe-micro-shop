# Mobile WEB 电商系统设计规范

## 一、设计价值观

- **以用户为中心**：所有设计决策以提升用户体验为核心，关注易用性、可访问性和情感共鸣。
- **简洁高效**：界面简洁、信息层级清晰，减少用户认知负担，提升操作效率。
- **包容与多元**：兼容不同用户群体，关注无障碍与多终端体验。
- **一致与灵活**：风格统一，组件规范，支持业务灵活扩展。
- **信任与安全**：强化信息安全感，保护用户隐私，建立平台信任。

## 二、设计原则

- **清晰直观**：信息表达直接，操作路径明确，避免歧义。
- **视觉层级**：通过色彩、排版、留白等手段突出重点，弱化次要信息。
- **反馈及时**：所有交互均有明确反馈，减少用户焦虑。
- **可用优先**：优先保证主要流程（如购物、支付、售后）的流畅与可用。
- **移动优先**：以移动端为主，兼顾多终端响应式体验。

## 三、品牌与色彩体系

- **主题色**：#19aecc（主色，贯穿品牌与高亮元素）
- **辅助色**：#FFB300（强调/警示）、#00C48F（成功）、#F53F3F（错误）、#86909C（中性色）
- **背景色**：#F7F8FA（主背景）、#FFFFFF（内容区）、#F2F3F5（分割/卡片）
- **文字色**：#1D2129（主文本）、#4E5969（次文本）、#86909C（辅助文本）、#C9CDD4（禁用/占位）
- **渐变与特殊色**：用于banner、按钮等场景，需与主题色协调
- **色彩使用规范**：主题色用于主按钮、链接、选中、品牌元素，辅助色用于状态提示，背景与文字色保证对比度与可读性

## 四、排版与字体

- **主字体**：系统默认字体（如"PingFang SC", "Helvetica Neue", Arial, sans-serif）
- **字号体系**：
  - 标题：20/18/16px（H1/H2/H3）
  - 正文：14/16px
  - 辅助/说明：12px
- **行高**：1.5-1.7倍字号，保证阅读舒适
- **字重**：标题加粗，正文常规，辅助信息细体
- **文本对齐**：左对齐为主，数字/金额右对齐
- **排版规范**：层级分明，留白适度，避免拥挤

## 五、图标与插画

- **图标风格**：线性、简洁、易识别，统一尺寸（24x24/20x20px）
- **图标库**：优先使用平台自有图标库，补充自定义时需风格统一
- **插画/空状态**：风格友好、轻量，传递品牌温度

## 六、组件规范

- **按钮**：主按钮（主题色）、次按钮（中性色）、禁用/加载/幽灵按钮
- **输入框**：圆角、清晰边框，聚焦高亮，支持错误/成功/禁用状态
- **下拉/选择器**：操作区分明，选中高亮，支持搜索
- **卡片**：商品、订单、活动等信息模块化展示，卡片有阴影/分割线
- **弹窗/抽屉**：用于重要操作确认或补充信息，支持手势关闭
- **标签/徽章**：用于状态、分类、促销等场景，色彩区分明显
- **导航栏/Tab**：底部主导航+顶部辅助导航，图标+文字组合
- **进度/步骤条**：用于订单、支付等流程展示，步骤清晰
- **消息/反馈**：Toast、气泡、提示条，反馈及时不打扰
- **社区/互动**：评论、点赞、分享、问答等组件风格统一

## 七、交互与动效

- **动效原则**：自然、克制，辅助理解流程与状态变化
- **按钮/卡片点击**：有按压/波纹反馈
- **加载/切换**：骨架屏、渐变加载，减少等待焦虑
- **页面切换**：平滑过渡，避免突兀
- **状态变化**：如购物车添加、支付成功等有正向动效

## 八、无障碍与可访问性

- **色彩对比**：主色与背景对比度≥4.5:1，保证弱视可读
- **文本可放大**：支持系统字体缩放
- **控件可聚焦**：所有可交互元素支持键盘/辅助技术聚焦
- **图片/图标**：提供alt文本或aria-label
- **动效可关闭**：为敏感用户提供动效开关

## 九、响应式与适配

- **移动优先**：适配主流手机屏幕（375-428px宽）
- **平板/桌面**：自适应布局，内容区最大宽度1200px
- **断点设计**：常用断点375/414/768/1024/1200px
- **触控友好**：控件尺寸≥44x44px，间距适中

## 十、品牌延展与视觉资产

- **Logo与品牌元素**：统一规范，适配深浅色背景
- **Banner/活动图**：主题色为主，风格活泼、吸引眼球
- **营销/促销元素**：如优惠券、标签、倒计时，色彩与动效突出

## 十一、文案与语言

- **风格**：简洁、友好、积极正向
- **提示语**：明确操作结果，避免歧义
- **多语言**：预留国际化能力，支持中英文切换

> 本设计规范为Mobile WEB电商系统全局适用，后续可根据业务与品牌发展持续优化补充。
