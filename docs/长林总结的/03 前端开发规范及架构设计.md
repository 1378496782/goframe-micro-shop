# Mobile WEB 电商系统前端开发规范及架构设计

## 一、技术选型与工程化基础

- **主框架**：Vue 3.x（Composition API，响应式、类型友好、生态完善）
- **组件库**：Varlet UI（Material Design 2/3 风格，移动端优先，支持主题定制、国际化、暗黑模式）
- **样式方案**：Tailwind CSS（原子化 CSS，极致灵活，支持主题扩展，适配移动端）
- **图标方案**：iconify/vue（支持数千种图标集，SVG 矢量，按需加载）
- **状态管理**：Pinia（Vue 官方推荐，轻量、类型友好、支持模块化）
- **路由管理**：Vue Router 4.x（支持动态路由、路由守卫、懒加载）
- **API 通信**：Axios（主流 HTTP 客户端，支持拦截器、全局配置、TS 支持）
- **表单校验**：VeeValidate 或自定义 hooks（结合 Varlet 表单组件）
- **国际化**：vue-i18n（支持多语言切换，适配设计规范）
- **无障碍支持**：结合 Varlet、Tailwind 及自定义 aria 方案，提升可访问性
- **工程化工具**：Vite（极速开发、按需编译、原生 ESM 支持）、ESLint + Prettier（代码规范）、Stylelint（样式规范）、Husky + lint-staged（提交前校验）
- **单元测试**：Vitest（Vite 生态，极速单测）、Vue Test Utils
- **持续集成/部署**：GitHub Actions（自动化测试、构建、部署）
- **Mock 数据**：Mock Service Worker（MSW，前后端并行开发）
- **性能分析**：Lighthouse、Vite 插件、Web Vitals

## 二、项目结构规范（细化版）

```tree
├── public                  # 公共资源（favicon、静态html等）
├── src
│   ├── assets              # 静态资源（图片、SVG、字体、媒体等）
│   │   ├── images
│   │   ├── icons
│   │   └── fonts
│   ├── components          # 通用基础组件（Button、Dialog、Icon等）
│   ├── modules             # 业务模块（商品、订单、社区等，含业务专属组件）
│   ├── pages               # 路由页面（按功能分目录，页面级组件）
│   ├── layouts             # 页面布局（如主布局、登录布局、空白布局等）
│   ├── router              # 路由配置（静态、动态、守卫等）
│   ├── store               # 状态管理（Pinia，按模块拆分）
│   ├── services            # API 请求与数据层（按业务模块拆分）
│   ├── hooks               # 通用 hooks（如 useUser、useCart、useRequest）
│   ├── utils               # 工具函数（格式化、校验、通用逻辑等）
│   ├── constants           # 全局常量（如枚举、配置项、正则等）
│   ├── enums               # 业务枚举类型
│   ├── types               # 全局/业务类型定义（如接口、数据结构）
│   ├── middleware          # 路由/请求中间件（如鉴权、日志等）
│   ├── locales             # 国际化资源（多语言文案）
│   ├── styles              # Tailwind 配置、全局样式、主题变量
│   ├── app.vue             # 根组件
│   └── main.js             # 入口文件
├── mock                    # Mock 数据与接口（MSW等）
├── tests                   # 单元与端到端测试（__tests__、e2e等）
├── .env*                   # 多环境配置文件（.env.development、.env.production等）
├── .husky                  # Git hooks
├── .vscode                 # 推荐插件与配置
├── README.md               # 项目说明文档
├── tailwind.config.js      # Tailwind CSS 配置
├── varlet.config.js        # Varlet 主题/配置（如有）
├── package.json            # 依赖与脚本
└── ...                     # 其他工程相关文件
```

> 各目录简要说明：
>
> - **public**：静态资源根目录，打包时原样输出。
> - **assets**：图片、SVG、字体等静态资源，按类型细分。
> - **components**：全局/通用组件，跨业务复用。
> - **modules**：业务模块目录，聚合相关页面、组件、store、service。
> - **pages**：路由页面，通常为业务入口或聚合页。
> - **layouts**：页面布局模板，便于多场景复用。
> - **router**：路由表、守卫、动态路由等。
> - **store**：Pinia 状态管理，按业务/功能拆分。
> - **services**：API 请求、数据处理，按模块拆分。
> - **hooks**：自定义 hooks，复用逻辑。
> - **utils**：工具函数库。
> - **constants/enums**：全局常量、枚举，便于统一管理。
> - **types**：TS 类型定义（如接口、数据结构），即使主项目用 JS 也建议类型文档化。
> - **middleware**：中间件逻辑（如鉴权、埋点、请求拦截等）。
> - **locales**：国际化文案资源。
> - **styles**：全局样式、Tailwind 配置、主题变量。
> - **mock**：本地/联调 mock 数据。
> - **tests**：单元、集成、端到端测试。
> - **.env***：多环境变量配置。
> - **.husky**：Git hooks 自动化。
> - **.vscode**：团队推荐 IDE 配置。
> - **README.md**：项目说明、开发约定。
> - **tailwind.config.js/varlet.config.js**：样式/主题配置。
> - **package.json**：依赖、脚本、元信息。

## 三、开发规范

### 1. 代码规范

- 遵循 ESLint（推荐 airbnb-base 或 vue/recommended）、Prettier 统一格式
- 组件命名采用 PascalCase，文件夹/文件名 kebab-case
- 业务逻辑与视图分离，hooks 复用逻辑
- 严格类型注释（JSDoc）
- 样式优先用 Tailwind CSS 原子类，必要时用 @apply 或自定义 class
- 禁止魔法数字、硬编码，常量集中管理

### 2. 组件开发

- 优先复用 Varlet UI 组件，二次封装统一风格
- 通用组件放 components，业务组件放 modules
- 组件 props 明确、事件命名规范（update:modelValue、onXxx）
- 支持 slots、可扩展性、无障碍（aria）
- 图标统一用 iconify/vue，按需引入

### 3. 路由与权限

- 路由按页面/业务模块分层，支持嵌套路由
- 路由守卫实现登录、权限、角色校验
- 支持路由懒加载、动态 import
- 登录后自动跳转回原操作页面

### 4. 状态管理

- Pinia 按模块拆分（如 user、cart、order、community）
- 状态持久化（如购物车、用户信息）用 localStorage/sessionStorage
- 只在必要时全局管理，避免过度依赖 store

### 5. API 与数据

- Axios 封装统一请求/响应拦截、错误处理、token 注入
- services 层按业务模块拆分，接口类型定义清晰
- 支持 mock 数据，前后端并行开发

### 6. 样式与主题

- Tailwind CSS 配合 Varlet 主题变量，支持暗色模式
- 全局样式统一管理，支持多主题切换
- 响应式断点、字体、间距等遵循设计规范

### 7. 国际化与无障碍

- vue-i18n 统一管理文案，支持中英文切换
- 组件/页面均需适配国际化
- 重点交互、表单、社区等场景加强无障碍支持

### 8. 测试与质量保障

- 单元测试（Vitest + Vue Test Utils）覆盖核心逻辑与组件
- 提交前自动 lint、测试，CI 阶段自动化校验

### 9. 性能与体验优化

- 路由/组件懒加载，首屏优化
- 图片压缩、SVG 雪碧图、骨架屏
- 关键数据缓存、接口防抖/节流
- Lighthouse/Web Vitals 持续监控

### 10. 安全与合规

- XSS/CSRF 防护，表单校验、token 安全
- 敏感信息脱敏、错误日志收集
- 遵循 GDPR/隐私合规要求

## 四、推荐开发工具与插件
