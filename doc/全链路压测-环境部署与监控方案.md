# 全链路压测环境部署与监控方案

## 1. 项目概述

本文档详细描述了shop-goframe微服务架构下的全链路压测环境部署与监控方案，旨在确保压测过程可控、可观察，同时不影响真实业务运行。

## 2. 压测环境部署架构

### 2.1 Locust分布式压测架构

```
+------------------+      +-------------------+      +-------------------+
|                  |      |                   |      |                   |
|  Locust Master   +----->+  Locust Worker 1  +----->+    被测系统      |
|                  |      |                   |      |    各微服务      |
+--------+---------+      +-------------------+      +-------------------+
         |
         |      +-------------------+
         |      |                   |
         +----->+  Locust Worker 2  |
                |                   |
                +-------------------+
```

### 2.2 硬件资源配置建议

| 角色 | 最低配置 | 推荐配置 | 数量 | 说明 |
|------|----------|----------|------|------|
| Locust Master | 4核8G | 8核16G | 1 | 负责协调整个压测过程 |
| Locust Worker | 8核16G | 16核32G | 3+ | 实际生成压测流量 |
| 监控服务器 | 4核8G | 8核16G | 1 | 部署Prometheus、Grafana等 |

## 3. 环境部署步骤

### 3.1 基础环境准备

```bash
# 所有机器安装Python3.8+
apt update
apt install -y python3 python3-pip

# 安装Locust
pip3 install locust==2.22.0

# 安装监控所需组件（在监控服务器上）
pip3 install prometheus-client grafana-api
```

### 3.2 Locust Master部署

1. 创建Master配置文件：

```python
# master_config.py
class MasterConfig:
    MASTER_HOST = "0.0.0.0"
    MASTER_PORT = 5557
    WEB_HOST = "0.0.0.0"
    WEB_PORT = 8089
    LOG_LEVEL = "INFO"
    LOG_FILE = "/logs/locust_master.log"
```

2. 启动Master节点：

```bash
locust -f /path/to/locustfile.py --master --master-bind-host=0.0.0.0 --master-bind-port=5557 -H http://target-service
```

### 3.3 Locust Worker部署

1. 复制相同的locustfile.py到所有Worker节点

2. 启动Worker节点：

```bash
locust -f /path/to/locustfile.py --worker --master-host=MASTER_IP --master-port=5557
```

### 3.4 容器化部署方案（可选）

使用Docker Compose快速部署Locust集群：

```yaml
# docker-compose-locust.yml
version: '3'
services:
  master:
    image: locustio/locust:2.22.0
    ports:
      - "8089:8089"
      - "5557:5557"
    volumes:
      - ./locustfile.py:/locust/locustfile.py
      - ./logs:/logs
    command: -f /locust/locustfile.py --master --master-bind-host=0.0.0.0

  worker1:
    image: locustio/locust:2.22.0
    volumes:
      - ./locustfile.py:/locust/locustfile.py
    command: -f /locust/locustfile.py --worker --master-host=master

  worker2:
    image: locustio/locust:2.22.0
    volumes:
      - ./locustfile.py:/locust/locustfile.py
    command: -f /locust/locustfile.py --worker --master-host=master
```

## 4. 监控系统设计

### 4.1 监控指标体系

#### 4.1.1 压测工具层面指标

- 并发用户数
- 请求总数/秒 (RPS)
- 响应时间（平均值、中位数、95%分位数、99%分位数）
- 错误率
- 每秒请求失败数

#### 4.1.2 应用服务层面指标

- CPU使用率
- 内存使用率
- GC次数和耗时
- 请求处理时间
- 数据库查询耗时
- 接口调用次数
- 错误日志数量

#### 4.1.3 基础架构层面指标

- 服务器资源使用率
- 网络吞吐量
- 数据库连接数
- Redis内存使用率
- RabbitMQ队列长度

### 4.3 自定义监控脚本

为Goframe应用添加Prometheus指标导出：

```go
// 在app/svc/metrics/metrics.go中添加
package metrics

import (
    "github.com/gogf/gf/v2/frame/g"
    "github.com/gogf/gf/v2/net/ghttp"
    "github.com/prometheus/client_golang/prometheus"
    "github.com/prometheus/client_golang/prometheus/promhttp"
)

// 定义监控指标
var (
    // 请求计数
    RequestCount = prometheus.NewCounterVec(
        prometheus.CounterOpts{
            Name: "app_requests_total",
            Help: "Total number of requests",
        },
        []string{"service", "method", "path", "status"},
    )
    
    // 请求延迟
    RequestLatency = prometheus.NewHistogramVec(
        prometheus.HistogramOpts{
            Name:    "app_request_duration_seconds",
            Help:    "Request duration in seconds",
            Buckets: prometheus.DefBuckets,
        },
        []string{"service", "method", "path"},
    )
)

// InitMetrics 初始化监控指标
func InitMetrics() {
    prometheus.MustRegister(RequestCount, RequestLatency)
}

// RegisterMetricsHandler 注册监控指标处理函数
func RegisterMetricsHandler(s *ghttp.Server) {
    s.BindHandler("/metrics", func(r *ghttp.Request) {
        promhttp.Handler().ServeHTTP(r.Response.Writer, r.Request)
    })
}

// 中间件：记录请求指标
func MetricsMiddleware(r *ghttp.Request) {
    startTime := time.Now()
    service := g.Cfg().GetString("service.name", "unknown")
    
    r.Middleware.Next()
    
    duration := time.Since(startTime).Seconds()
    RequestCount.WithLabelValues(
        service,
        r.Method,
        r.URL.Path,
        strconv.Itoa(r.Response.Status),
    ).Inc()
    
    RequestLatency.WithLabelValues(
        service,
        r.Method,
        r.URL.Path,
    ).Observe(duration)
}
```

## 5. 告警机制设计

### 5.1 Prometheus Alertmanager配置

1. 安装Alertmanager：

```bash
wget https://github.com/prometheus/alertmanager/releases/download/v0.25.0/alertmanager-0.25.0.linux-amd64.tar.gz
tar -xzf alertmanager-0.25.0.linux-amd64.tar.gz
```

2. 配置告警规则：

```yaml
# alerts.yml
groups:
- name: locust_alerts
  rules:
  - alert: HighErrorRate
    expr: sum(rate(locust_requests_failed_total[5m])) / sum(rate(locust_requests_total[5m])) > 0.05
    for: 1m
    labels:
      severity: critical
    annotations:
      summary: "High error rate detected"
      description: "Error rate is {{ $value | humanizePercentage }}"
  
  - alert: HighResponseTime
    expr: histogram_quantile(0.95, sum(rate(locust_request_duration_seconds_bucket[5m])) by (le)) > 1
    for: 1m
    labels:
      severity: warning
    annotations:
      summary: "High response time detected"
      description: "95th percentile response time is {{ $value }}s"

- name: service_alerts
  rules:
  - alert: HighCPULoad
    expr: sum(rate(process_cpu_seconds_total[5m])) by (instance) / count(node_cpu_seconds_total{mode="idle"}) by (instance) * 100 > 80
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "High CPU load detected"
      description: "CPU load on {{ $labels.instance }} is {{ $value | humanizePercentage }}"
  
  - alert: HighMemoryUsage
    expr: (node_memory_MemTotal_bytes - node_memory_MemAvailable_bytes) / node_memory_MemTotal_bytes * 100 > 85
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "High memory usage detected"
      description: "Memory usage on {{ $labels.instance }} is {{ $value | humanizePercentage }}"
```

## 6. 资源隔离策略

### 6.1 网络隔离

- 使用VLAN或子网隔离压测环境
- 配置防火墙规则，限制压测流量范围
- 为压测环境配置独立的负载均衡器

### 6.2 服务限流

```go
// 在app/middleware/rate_limit.go中添加压测流量限流
package middleware

import (
    "github.com/gogf/gf/v2/frame/g"
    "github.com/gogf/gf/v2/net/ghttp"
    "github.com/gogf/gf/v2/util/gconv"
    "golang.org/x/time/rate"
    "sync"
    "time"
)

var (
    // 普通流量限流器
    normalLimiter = rate.NewLimiter(rate.Limit(1000), 100)
    // 压测流量限流器
    testLimiter = rate.NewLimiter(rate.Limit(500), 50)
    limiterLock sync.Mutex
)

// RateLimitMiddleware 实现流量限流
func RateLimitMiddleware(r *ghttp.Request) {
    // 检查是否是压测流量
    isTestTraffic := r.GetHeader("X-Test-Traffic") == "true"
    
    var limiter *rate.Limiter
    if isTestTraffic {
        limiter = testLimiter
    } else {
        limiter = normalLimiter
    }
    
    // 尝试获取令牌
    if !limiter.Allow() {
        r.Response.Status = 429
        r.Response.WriteJson(g.Map{
            "code": 429,
            "msg":  "Too many requests",
        })
        r.ExitAll()
        return
    }
    
    r.Middleware.Next()
}
```

## 8. 常见问题及解决方案

### 8.1 Locust工作节点连接问题

**症状**：Worker节点无法连接到Master节点
**解决方案**：
- 检查网络防火墙设置
- 确认Master节点的IP地址配置正确
- 验证端口5557是否开放

### 8.2 监控数据采集不完整

**症状**：Grafana面板部分指标缺失
**解决方案**：
- 检查Prometheus配置是否包含所有目标
- 验证服务的/metrics端点是否可访问
- 确认Prometheus抓取间隔设置合理

### 8.3 压测过程中系统异常

**症状**：压测过程中系统出现崩溃或异常
**解决方案**：
- 立即停止压测
- 检查系统日志查找错误原因
- 调整压测参数，降低并发或增加资源
- 检查数据隔离是否有效
