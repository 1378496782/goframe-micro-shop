# 项目封版与部署上线

这份文档用于把当前 GoFrame 微服务商城阶段性收口，优先服务面试和项目展示。

当前策略不是放弃秒杀高并发，而是先把已经完成的订单一致性、MQ、Outbox、CDC、Redis 缓存、互动模块等能力打包成可部署、可验证、可写进简历的项目版本。秒杀、K8s、链路追踪、压测等内容后续继续补实战。

## 本次封版目标

本次先交付一个 Docker Compose 版本：

```text
用户端 H5 网关 + 业务微服务 + 搜索服务 + GFast 管理后台 + 基础中间件
```

目标不是追求“完整线上生产级”，而是做到：

- 后端主链路能容器化启动
- 管理后台前端 `dist` 能由 `manage-service` 托管访问
- H5 网关能访问商品、购物车、订单、搜索、互动接口
- MySQL / Redis / RabbitMQ / Elasticsearch / etcd 依赖关系清楚
- 能用日志、数据库、RabbitMQ 控制台、ES 查询验证关键链路
- 能把项目能力整理成简历和面试讲法

## 本次上线范围

### 后端主链路

- 用户：登录、注册、收货地址
- 商品：列表、详情、分类、库存、Redis 商品详情缓存
- 购物车：加购、更新数量、列表汇总
- 订单：预览、创建、取消、支付成功回调模拟、销量增加
- 一致性：MQ 异步消费、消费幂等、Outbox 本地消息表、补偿任务
- 搜索：MySQL 商品数据通过 CDC 同步 ES，H5 网关代理 `/frontend/search/goods`
- 互动：评论、回复、点赞、收藏、用户隔离、评论点赞计数、Redis 旁路缓存、定时校准、分布式锁

### 前端与管理后台

- 用户端先以 H5 网关接口可调通为主，小程序 / H5 页面逐步联调
- 管理端本次优先采用 GFast 管理后台：
  - 后端：`../shop-goframe-micro-manage`
  - 前端：`../shop-goframe-micro-service-manage-ui-gfast/dist`
  - 容器：`manage-service`

### 基础设施

- MySQL 8.0
- Redis 7
- RabbitMQ 4.1 管理端 + 延迟消息插件
- etcd 3.5
- Elasticsearch 8.11 + IK 分词
- Kibana
- MySQL 每日备份容器

## 暂不纳入本次上线

这些内容可以讲设计思路，但不要在简历里写成“已完整落地”：

- 秒杀独立服务
- Redis Lua 秒杀扣库存真实下单链路
- Redis 主写点赞 + dirty set 聚合落库
- K8s 云端部署、HPA、灰度发布
- Prometheus / Grafana / Jaeger / Loki 完整观测闭环
- 真实微信支付生产回调
- 管理端 `gateway-admin + admin-service` 这套微服务网关路径

本次管理后台先收敛到 GFast `manage-service`，避免两套管理端同时上线导致范围变大。

## 当前生产编排

生产编排入口：

```bash
docker compose -f docker-compose.prod.yml config --quiet
docker compose -f docker-compose.prod.yml build
docker compose -f docker-compose.prod.yml up -d
```

核心服务：

| 服务 | 端口 | 用途 |
| --- | --- | --- |
| `gateway-h5` | `8199` | 用户端 HTTP 网关 |
| `manage-service` | `8808` | GFast 管理后台和静态前端 |
| `gateway-resource` | `8399` | 文件上传 / 资源网关 |
| `search-service` | `8499` | ES 商品搜索 |
| `user-service` | `31001` | 用户 gRPC 服务 |
| `interaction-service` | `31002` | 评论 / 点赞 / 收藏 gRPC 服务 |
| `goods-service` | `31004` | 商品 / 购物车 gRPC 服务 |
| `order-service` | `31005` | 订单 / 支付 / MQ 消费 gRPC 服务 |
| `banner-service` | `31006` | 轮播图 gRPC 服务 |

本次已确认并修正的生产依赖：

- `goods-service` 启动时会创建 RabbitMQ 消费者，所以必须等待 `rabbitmq` 健康。
- `interaction-service` 生产配置会连接 Redis，点赞计数校准也依赖 Redis，所以必须等待 `redis` 健康。

## 上线前检查清单

### 1. 工程目录

生产 Compose 使用相邻目录：

```text
goframe-micro-shop/
  shop-goframe-micro-service-refacotor/
  shop-goframe-micro-manage/
  shop-goframe-micro-service-manage-ui-gfast/
```

不要单独移动 `shop-goframe-micro-service-refacotor`，否则 `manage-service` 构建上下文和前端 `dist` 挂载会失效。

### 2. 前端构建

管理后台前端必须先生成 `dist`：

```bash
cd ../shop-goframe-micro-service-manage-ui-gfast
npm run build
```

Compose 会把它挂载到：

```text
../shop-goframe-micro-service-manage-ui-gfast/dist:/app/resource/public
```

### 3. 生产配置

上线前必须检查：

- `app/*/manifest/config/config.prod.yaml` 里的 MySQL / Redis / RabbitMQ / etcd / ES 地址是否使用 Docker 服务名
- `../shop-goframe-micro-manage/manifest/config/config.prod.yaml` 是否使用 `serverRoot: "resource/public"`
- 管理后台前端 `.env.production` 的 `VITE_API_URL` 是否符合最终访问方式
- 不要把 RabbitMQ、Kibana、Elasticsearch 管理端直接暴露给公网

当前仓库里仍有硬编码密钥和三方配置。真正公网部署前要做一次密钥轮换，并把敏感配置迁出仓库，至少用服务器环境变量或单独的 `.env` 管理。

### 4. 外部访问入口

本次推荐只暴露：

- 用户端：反向代理到 `gateway-h5:8199`
- 管理端：反向代理到 `manage-service:8808`
- 资源接口：按需要反向代理到 `gateway-resource:8399`

内部服务端口、MySQL、Redis、RabbitMQ、ES、Kibana 默认不公开。

## 功能收尾验收

### H5 网关

最小冒烟链路：

```text
商品列表 -> 商品详情 -> 加购物车 -> 订单预览 -> 创建订单 -> 取消订单
```

验证点：

- 页面 / API 返回成功
- `order_info` 状态正确
- 库存扣减和恢复正确
- 日志里能看到 gateway-h5 -> gRPC 服务调用

### MQ / Outbox

最小冒烟链路：

```text
支付成功模拟 -> 写入 order_outbox_message -> Outbox 中继投递 RabbitMQ -> 消费者增加销量
```

验证点：

- `order_outbox_message.status` 从 `pending` 变成 `sent`
- RabbitMQ 队列无持续堆积
- `goods_info.sales` 增加一次
- 重复事件不会重复加销量

### 搜索

最小冒烟链路：

```text
新增 / 更新商品 -> CDC 消费 Binlog -> 写入 ES -> H5 搜索命中
```

验证点：

- `search-service` 日志能看到 ES 写入
- `/frontend/search/goods?keyword=...` 返回预期商品
- 中文搜索命中 `name / brand / tags / detail_info`

### 互动模块

最小冒烟链路：

```text
创建一级评论 -> 创建回复 -> 查询评论列表 -> 点赞评论 -> 取消点赞
```

验证点：

- 回复挂在对应一级评论的 `replies` 下
- `reply_count` 只维护在一级评论上
- 重复点赞 / 重复取消幂等成功
- `comment_info.like_count` 不为负
- Redis 点赞计数 miss 时能回落 MySQL
- 校准任务日志正常，且多实例时只有一个实例持有分布式锁执行

### 管理后台

最小冒烟链路：

```text
访问管理后台 -> 登录 -> 商品 / 订单 / 用户数据页可打开
```

验证点：

- `manage-service` 能返回前端首页
- API 请求能连上 MySQL / Redis
- 静态资源路径不 404
- 上传目录挂载可用

## 简历可写能力

当前可以写成“已实现 / 已落地”的内容：

- 基于 GoFrame + gRPC + HTTP 网关实现微服务电商系统，拆分用户、商品、订单、互动、搜索等服务。
- 订单链路引入状态机、补偿任务、RabbitMQ 异步消费和 Outbox 本地消息表，解决支付成功加销量、取消恢复库存等最终一致性问题。
- 商品详情使用 Redis Cache Aside、空值缓存、TTL 随机化和互斥重建，降低热点商品读请求对 MySQL 的压力。
- 商品搜索使用 Elasticsearch 多字段检索，并通过 Binlog / CDC 将 MySQL 商品变更同步到 ES。
- 互动模块实现多级评论建模、点赞幂等、MySQL 冗余计数、Redis 旁路缓存、定时校准和 Redis 分布式锁。
- 使用 Docker Compose 编排 MySQL、Redis、RabbitMQ、etcd、Elasticsearch、Kibana 和业务服务，完成本地生产形态部署。

可以写成“设计过 / 了解 / 后续可扩展”的内容：

- 秒杀高并发场景下的 Redis Lua 原子扣库存、限流、异步下单和库存最终一致性。
- Redis 主写点赞计数、dirty set 聚合落库、Bitmap / 分片 Set 热点优化。
- K8s 部署、HPA 扩缩容、链路追踪、监控告警和压测验证。
