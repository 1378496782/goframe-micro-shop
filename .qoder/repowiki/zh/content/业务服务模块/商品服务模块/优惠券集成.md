## 简介
本文件面向“优惠券集成”功能，系统化阐述优惠券在商品购买流程中的应用，包括优惠券查询、使用规则验证、优惠金额计算、发放与使用记录追踪、统计分析等能力。文档覆盖数据模型设计（优惠券类型、使用条件、有效期、适用范围）、与商品价格的计算关系、多张优惠券的叠加规则与限制、以及API接口与配置参数说明，并通过序列图与类图展示关键流程与代码结构。

## 项目结构
优惠券相关能力由多个模块协同实现：
- 商品侧（goods）：提供优惠券基础信息管理（创建、查询、更新、删除），以及用户持有优惠券的管理。
- 用户侧（user）：提供优惠券发放服务接口（当前未实现，预留扩展）。
- 订单侧（order）：在下单流程中对优惠券进行校验与分摊计算，确保订单金额与优惠一致。

### 订单侧优惠券校验与分摊
- 校验逻辑
  - 订单总价=商品总价
  - 实际支付=订单总价-总优惠
  - 总优惠≥各明细优惠之和
- 分摊逻辑
  - 已预分配的明细不再参与分摊
  - 未预分配明细按价格比例分摊剩余优惠，最后一条明细补足差额

```mermaid
flowchart TD
Start(["进入下单校验"]) --> SumGoods["计算商品总价"]
SumGoods --> SumCoupon["计算明细优惠之和"]
SumCoupon --> CheckTotal{"订单总价=商品总价？"}
CheckTotal --> |否| Err1["返回错误：总价不一致"]
CheckTotal --> |是| CheckActual{"实际支付=总价-总优惠？"}
CheckActual --> |否| Err2["返回错误：实际支付不正确"]
CheckActual --> |是| CheckSumCoupon{"总优惠≥明细优惠之和？"}
CheckSumCoupon --> |否| Err3["返回错误：总优惠不足"]
CheckSumCoupon --> |是| PreAssign["收集已预分配明细与可分配明细"]
PreAssign --> CalcRemain["计算剩余可分摊优惠"]
CalcRemain --> HasRemain{"有剩余且存在可分配明细？"}
HasRemain --> |否| Done["完成"]
HasRemain --> |是| Allocate["按价格比例分摊剩余优惠"]
Allocate --> LastOne["最后一条补足差额"]
LastOne --> Done
Err1 --> End(["结束"])
Err2 --> End
Err3 --> End
Done --> End
```

## 性能考虑
- 分页查询：列表接口支持分页，避免一次性加载大量数据
- 时间字段转换：控制器中对时间字段进行安全转换，减少序列化开销
- 优惠分摊：按价格比例分摊时使用大整型计算，避免溢出并保证精度
- 未实现接口：用户模块的发放接口当前返回未实现，避免无效调用带来的资源消耗

## 故障排查指南
- 数据库操作错误
  - 控制器统一捕获数据库错误并返回标准化错误码
  - 建议检查DAO层SQL执行与表结构一致性
- 订单校验失败
  - 检查商品明细总价与订单总价是否一致
  - 检查总优惠与明细优惠之和的关系
