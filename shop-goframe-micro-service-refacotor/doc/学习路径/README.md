# 学习路径目录

这个目录承接后续学习主线。

旧文档 `../学习路径-商品列表.md` 是早期综合笔记，内容很全，但目录已经混杂。后续新增学习记录优先写到这里。

## 阅读顺序

1. `00-学习总路线.md`
   - 唯一维护当前进度、下一步和完整主线。

2. `04-订单模块.md`
   - 记录订单链路：预览、创建、扣库存、取消、支付、销量、补偿。

3. `08-互动模块.md`
   - 记录评论、回复、点赞、计数一致性。

4. `09-秒杀高并发专题.md`
   - 记录秒杀活动、Redis Lua 扣库存、异步下单、限流和最终一致性。

## 记录原则

每轮学习只记录三类内容：

- 这轮解决了什么问题
- 核心设计为什么这么做
- API Fox / 数据库 / 日志怎么验证

不要把所有临时思考都塞到一个文件里。以后按模块拆开记，主线写在总路线里。

以后只有 `00-学习总路线.md` 写“当前下一步”。模块文档只记录本模块已经学过什么、为什么这么做、怎么验证。

---

你之后每轮结束，可以固定这样跟我说：

```
这一轮我写完了，按学习路径帮我 review。
如果没问题：
1. 把重要内容记录到对应模块文档
2. 更新 00-学习总路线.md 的当前下一步
3. 告诉我下一轮要做什么
```

更短一点也可以：

```
这一轮完成了，review + 记文档 + 推进下一步
```

如果你只是想让我检查代码，不想改文档，就说：

```
我写完了，先只 review，不要改文档
```

如果你还没写，只是想知道下一步：

```
按 00-学习总路线.md 继续，下一步我该做什么？
```

我之后会默认遵守这个规则：

```
README.md 不动
00-学习总路线.md 负责当前下一步
模块文档只记录关键学习沉淀
```

这样我们就不会每次到处改文档了。
